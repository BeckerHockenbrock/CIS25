#include <iostream>
#include <string>

using namespace std;

class Product{
private:

    int id;
    string name;
    double price;

public:

    Product(int i, string n, double p){

        id = i;
        name = n;
        price = p;

        cout << "Made the " << name << " object." << endl;

    }

    ~Product(){
        cout << name << " was descontructed." << endl;
    }

    void printDetails(){

        cout << "Id: " << id << endl;
        cout << "Name: " << name << endl;
        cout << "Price: " << price << endl;

    }



};

int main(){

    Product p = Product(100, "basketball", 10.50);
    p.printDetails();
    return 0;


}

Knowlege check:

1: In my Product program, the constructor is used when the Product object is created.
It initializes the product’s ID, name, and price, then prints a message confirming that the object was made.
The destructor runs automatically when the object reaches the end of its lifetime, which happens when main() ends in this program.
Right now it prints a message; in a larger project, a destructor can also release the memory the object owns.

2: The Product class, constructor, and destructor are all connected to the same object type.
The class is the blueprint for Product objects.
The constructor, Product(...), and destructor, ~Product(), are special member functions of that class
Both use the class name and have no return type. The constructor handles object setup, while the destructor handles object cleanup.
